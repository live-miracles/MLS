Stream Names

You can assign names for the streams.

Set Outputs [file]

The nginx server receives the feed and sends it to a multiple outputs, such as when streaming on the YouTube or Facebook. You can use this interface to add such destination.
A few important points:

  • "Name" field should not contain any spaces. Please use underscores "_".
  • "Name" field should be unique within the stream.
  • When streaming locally between streams use the localhost IP: 127.0.0.1
  • Select "Vertical" when streaming to Instagram. It will rotate the video by 90 degrees clockwise and encode into 720p resolution. Server will need around 3 vCPU for each vertical stream.
  • If you keep the source resolution for all destinations the server is capable of streaming 100+ outs with just 10 vCPU cores.
  • If a login/password has to be added, enter the details in the format:
    rtmp://server/streamkey-+-+flashver=FMLE/3.020(compatible;20FMSc/1.0)-+-+pubUser=login-+-+pubPasswd=password


Add Stream Config [file]

You can apply an encoding to the stream. When encoding 1080p video into 720p it takes about 2.5 vCPU per stream.
Select "None" if you don't want any encodings then it will not take much CPU resources since it will be be only copying the data and not processing it.

Add Audio Config [file]

You can stream multichannel audio to nginx using stream1080 main or backup. Then you can split these audio channels into streams. In this audio config you can choose which channel you want to give to each stream.

Add/Remove Schedule [file]

Use it to schedule a cron job. You can add multiple jobs with the same name, but if you click the remove button, it will remove all of them at once.
